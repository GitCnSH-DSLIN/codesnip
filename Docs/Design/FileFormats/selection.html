File Format Documentation

Selection Files

Introduction

CodeSnip uses this file format when it saves the current selection of snippets to disk. The files use a custom format.

Encoding

This is a simple plain text file format encoded as UTF-8 with byte order mark.

File Format

The file is introduced by a header, which must occupy the first line. This line must be: 

► CodeSnip Selections v2 ◄

No blank lines or white space may precede the header line.

Immediately following the header line comes a list of snippet IDs for the snippets in the selection. There is one snippet per line. Snippets IDs must be well formed, but do not have to exist in the database. There must be at least one snippet line.

Blanks lines are permitted anywhere after the header line. They are ignored.

Lines are separated by a CR / LF pair.

NOTE: The ► and ◄ characters were chosen for the header line because they encode in a unique way in UTF-8 so that ANSI and UTF-16 format files will not encode them correctly. This provides a second way to check for the correct file format in addition to the file's byte order mark.

Program Support

This file format is supported by CodeSnip 5.

CodeSnip 4 used a similar but incompatible file format.
